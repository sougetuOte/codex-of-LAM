# UI デザインガイドスキル

PLANNING フェーズでの UI/UX 仕様策定時に、設計原則レベルのチェックリストを提供する。
フレームワーク非依存。実装の詳細ではなく「設計段階で考慮すべきこと」に焦点を当てる。

## 適用条件

以下のいずれかに該当する場合、このスキルを適用する:
- `docs/specs/ui-*.md` の新規作成
- UI/UX 設計、画面設計、コンポーネント設計の議論
- フロントエンド仕様の策定

## 1. アクセシビリティ (WCAG 2.1 AA)

仕様書に以下を明記しているか確認する:

| チェック項目 | 基準値 | 説明 |
|:---|:---|:---|
| 色コントラスト比 | 4.5:1 以上（通常テキスト）、3:1 以上（大テキスト） | 色だけに依存しない情報伝達 |
| タッチターゲット | 最小 44x44 CSS px（推奨 48x48） | モバイルでの操作性 |
| キーボード操作 | 全機能がキーボードのみで操作可能 | Tab 順序が視覚順序と一致 |
| フォーカス表示 | 可視的なフォーカスリング | `:focus-visible` の活用 |
| 代替テキスト | 意味のある画像に alt テキスト | 装飾画像は `alt=""` |
| 見出し階層 | h1 → h2 → h3 の論理順序 | レベルのスキップ禁止 |
| フォームラベル | 全入力要素に `<label>` 関連付け | placeholder はラベル代替にならない |
| モーション | `prefers-reduced-motion` 対応 | アニメーション無効化オプション |

## 2. 状態設計 (UI States)

各画面・コンポーネントについて以下の 5 状態を設計しているか確認する:

| 状態 | 説明 | よくある見落とし |
|:---|:---|:---|
| **Empty** | データなし | 「データがありません」だけでは不十分。次のアクションを案内 |
| **Loading** | 読み込み中 | スケルトンスクリーン or スピナー。3秒超は進捗表示 |
| **Error** | エラー発生 | 何が起きたか + どうすればよいか。技術用語は避ける |
| **Success** | 成功 | フィードバックの表示時間とタイミング |
| **Partial** | 一部成功/一部失敗 | バッチ操作時に特に重要 |

## 3. レスポンシブ設計

仕様書にブレークポイント戦略を明記しているか確認する:

| 観点 | 設計指針 |
|:---|:---|
| ブレークポイント | モバイルファースト。最低 3 段階（mobile / tablet / desktop） |
| レイアウト | リフロー可能な設計。横スクロール禁止（テーブル等の例外を除く） |
| 画像 | レスポンシブ画像（srcset / picture）。WebP/AVIF 対応検討 |
| タイポグラフィ | `clamp()` によるフルイドタイポグラフィ or メディアクエリ段階調整 |
| ナビゲーション | モバイルでのナビゲーションパターン（ハンバーガー、ボトムナビ等） |

## 4. フォーム UX

| チェック項目 | 設計指針 |
|:---|:---|
| バリデーション | インラインバリデーション。入力離脱時（onBlur）に検証 |
| エラー表示 | エラーメッセージはフィールド直下に表示。赤色 + アイコン |
| 入力補助 | 適切な `type` 属性（email, tel, url）でモバイルキーボード最適化 |
| 自動保存 | 長いフォームでは途中保存を検討 |
| 確認画面 | 不可逆操作（削除等）には確認ステップ |
| 進捗表示 | 複数ステップのフォームにはプログレスバー |

## 5. パフォーマンス意識

仕様段階で考慮すべきパフォーマンス要因:

| 指標 | 目標値 | 設計への影響 |
|:---|:---|:---|
| LCP (Largest Contentful Paint) | 2.5 秒以下 | メインコンテンツの初期表示戦略 |
| CLS (Cumulative Layout Shift) | 0.1 以下 | 画像・広告の寸法事前指定 |
| 初期バンドルサイズ | 必要最小限 | コード分割・遅延読込の設計 |

## 公式ツール参照

実装段階では以下のツールの併用を推奨:

- **[frontend-design plugin](https://github.com/anthropics/claude-code/blob/main/plugins/frontend-design/skills/frontend-design/SKILL.md)**: Anthropic 公式。タイポグラフィ、カラー、モーション、レイアウトの実装ガイド
- **[ui-ux-pro-max-skill](https://github.com/nextlevelbuilder/ui-ux-pro-max-skill)**: 50+スタイル、97カラーパレット、WCAG AA 準拠チェック
- **[Web Interface Guidelines](https://www.webinterfaceguidelines.com/)**: Vercel による包括的な UI ガイドライン
