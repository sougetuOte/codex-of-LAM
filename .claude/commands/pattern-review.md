# /pattern-review - TDD パターン審査

> **MVP 制限**: 現在はパターンログ（`.claude/tdd-patterns.log`）の閲覧と手動審査のみ対応。
> ルール候補ファイル（`draft-*.md`）の自動生成は完全実装で対応予定。

## 概要

PostToolUse hook が記録した TDD 失敗パターンを確認し、閾値到達のルール候補を承認・却下する。

## Phase 1: パターン棚卸し

1. `.claude/tdd-patterns.log` を読み込み、記録されたパターンを一覧表示
2. `docs/artifacts/tdd-patterns/` 配下のパターンファイルがあれば参照（MVP では手動作成。完全実装で PostToolUse hook が自動生成）
3. 各パターンの観測回数を集計

```
=== TDD パターン一覧 ===

| # | パターン | 観測回数 | 最終観測 | ステータス |
|---|---------|---------|---------|-----------|
| 1 | [パターン名] | 2 | YYYY-MM-DD | 候補生成済み |
| 2 | [パターン名] | 2 | YYYY-MM-DD | 監視中 |
| 3 | [パターン名] | 1 | YYYY-MM-DD | 記録のみ |
```

## Phase 2: ルール候補の確認

閾値（2回）に到達したパターンについて:

1. `.claude/rules/auto-generated/draft-*.md` を読み込み（MVP では手動作成。完全実装で信頼度モデルが自動生成）
2. 各候補の根拠パターンと推奨ルールを表示
3. ユーザーに承認・却下を求める

```
=== ルール候補 ===

[Draft-001] パターン名: [名前]
観測回数: 2
根拠:
  - YYYY-MM-DD: src/xxx.py で [失敗内容]
  - YYYY-MM-DD: src/xxx.py で [失敗内容]

推奨ルール:
  「XXX のような変更を行う際は YYY に注意すること」

承認しますか？（承認 / 却下 / 保留）
```

## Phase 3: 承認処理

- **承認**: `draft-NNN.md` → `rule-NNN.md` にリネーム。ステータスを `approved` に変更
- **却下**: `draft-NNN.md` を削除。ステータスを `rejected` として記録
- **保留**: 変更なし。次回 `/pattern-review` で再表示

## Phase 4: 寿命チェック

承認済みルール（`rule-*.md`）のうち、90日以上未使用のものを通知:

```
=== 寿命チェック ===

[Rule-003] 最終マッチ: YYYY-MM-DD（XX日前）
  → 棚卸し対象。削除しますか？（削除 / 維持）
```

削除は **PM級**（人間承認必須）。

## 権限等級

- パターンの閲覧: **PG級**
- ルール候補の承認・却下: **PM級**
- 承認済みルールの削除: **PM級**
