# Wave Plan（Wave 計画策定）

次の Wave で実装するタスクを選定し、実行順序と依存関係を整理する。
PLANNING フェーズの一部として、BUILDING 開始前に実施する。

## 引数

- `/wave-plan` — 次 Wave を自動計画（デフォルト）
- `/wave-plan N` — Wave N を計画

## 実行ステップ

### Step 1: 現状把握

1. `docs/specs/` および `docs/tasks/` 配下のタスク定義を読み込む
2. `SESSION_STATE.md` で完了済み Wave を確認
3. git log で直近の Wave のコミット履歴を確認

```
--- Wave Plan: 現状 ---
完了: Wave 1〜N（T-01〜T-XX）
残タスク: X件
```

### Step 2: タスク選定

以下の基準で次 Wave のタスクを選定（1 Wave = 1-4 タスク、推奨上限 4）:

#### 選定基準（優先順）

1. **依存関係**: 前提タスクが完了しているもの
2. **統合リスク**: 相互に影響するタスクは同じ Wave にまとめる
3. **サイズバランス**: 1 Wave の合計が L+S 程度になるよう調整
4. **クリティカルパス**: Phase 完了に必要な最短経路を優先

#### 除外基準

- 前提タスクが未完了
- 後続 Phase に明示的に据え置かれたもの
- 外部依存でブロックされているもの

### Step 3: 実行順序の決定

選定したタスクの実行順序を決定:

```markdown
## Wave N 計画

| 順序 | タスク | 規模 | 依存 | 概要 |
|:----:|:-------|:----:|:-----|:-----|
| 1 | T-XX | M | なし | [概要] |
| 2 | T-YY | S | T-XX | [概要] |
| 3 | T-ZZ | S | なし | [概要] |

### 並列可能性
- T-XX と T-ZZ は並列実行可能
- T-YY は T-XX 完了後に開始

### Wave 完了条件
- [ ] 全タスクのテストが PASSED
- [ ] lint チェッククリーン
- [ ] 仕様書同期済み
- [ ] `/full-review` による監査完了
- [ ] `/ship` によるコミット完了
```

### 構造化思考の活用

Wave 構成の判断で以下のいずれかに該当する場合は `/magi` の使用を検討する:
- 判断ポイントが 2 つ以上（複数の Wave 割り当て案がある）
- 影響レイヤーが 3 つ以上（複数モジュールにまたがるタスク群）
- 有効な選択肢が 3 つ以上（並列化の組み合わせが複数）

### Step 4: リスク確認

```markdown
### リスク評価
| リスク | 影響 | 対策 |
|:-------|:-----|:-----|
| [リスク1] | [影響] | [対策] |
```

### Step 5: ユーザー確認

計画をユーザーに提示し、承認を求める:

```
--- Wave N 計画 ---
タスク: T-XX, T-YY, T-ZZ（計 X タスク）
推定規模: M + S + S
リスク: [あれば/なし]

この計画で進めますか？
1. 承認（`/building` で開始）
2. タスク変更（入れ替え・追加・除外）
3. 詳細確認（特定タスクの仕様を確認）
```

## 注意事項

- Wave 計画は PLANNING フェーズの一部として実施
- タスクの追加・変更は仕様書の存在を前提とする
- 1 Wave が 4 タスクを超える場合は分割を検討
- 大規模タスク（L）は単独 Wave を推奨
