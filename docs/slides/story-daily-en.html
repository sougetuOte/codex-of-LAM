Permission Levels — Automating Decisions

PG Level: Auto-fix, no report

AI organizes imports → auto-fix, no report

"Fixes that don't need human involvement"

SE Level: Report after fix

AI improves error handling → report after fix

"Involves technical judgment, but doesn't affect specs"

PM Level: Request user approval

Want to change API response format → request user approval

"Changes that affect specs are decided by humans"

Afternoon — AUDITING review

You

Audit src/auth/ with the review pane and focused verification

Living Architect

I'll separate the review scope, worker-assisted checks, and verification commands.

iter 1: 5 found → 5 fixed → re-scan
iter 2: 1 found → 1 fixed → re-scan
iter 3: 0 found → ✅ Green State

Automation can gather evidence, while the final gate remains visible to humans.

Green State: 5 Conditions

  • G1: All tests pass
  • G2: Zero lint errors
  • G3: Zero actionable issues
  • G4: Zero spec deviations
  • G5: Security check passed

Important: Green State means the scan finds 0 issues, not that issues were fixed

Context Management

Problem: Long work sessions exhaust the context window

1. Monitor context remaining via StatusLine at all times

2. AI recommends quick-save when below 20%

3. quick-save → lightweight save to SESSION_STATE.md (3-4% cost)

4. quick-load in a new session → instant recovery
