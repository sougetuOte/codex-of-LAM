PLANNING

Allowed: Design document output

Prohibited: Code generation

BUILDING

Required: Spec review, strict TDD

Prohibited: Implementation without specs

AUDITING

Allowed: PG/SE-level fixes

Prohibited: Direct PM-level changes (specs/architecture)

Core Principles

Active Retrieval

Always verify by reading files. Never assume you "remember" correctly.

Zero-Regression

Simulate impact before making changes. Verify ripple effects to the most distant modules.

Living Documentation

Implementation and documentation are an inseparable unit. Code-only changes are not allowed.

Hierarchy of Truth

User Intent > Architecture > Specs > Code. When in doubt, follow this order.
