Cycle 2 ルールが効く

  • 新しい作業ルールを AGENTS.md / workflow / skill に反映
  • 同じ失敗が発生しなくなる
  • 開発速度が向上

Cycle 1 の失敗経験が、Cycle 2 の自動ガードに変わった

ルール寿命管理

  • 各ルールに last_matched 日付を記録
  • 90日以上未使用のルールを quick-save 時の記録で棚卸し通知
  • 削除は PM級（人間承認必須）

ルールは増え続けるのではなく、新陳代謝する

まとめ

Living Architect Model は、あなたのプロジェクトと共に成長する。

失敗から学び、ルールを生成し、品質を自律的に高めていく。

それが "Living" の意味だ。
