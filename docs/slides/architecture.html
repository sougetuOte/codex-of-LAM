全てのツール操作を監視し、プロジェクトを守る3層の防御

PreToolUse — 門番

ファイルパスベースの権限チェック

  • docs/specs/, docs/adr/, AGENTS.md への変更 → PM級（承認必須）
  • src/ への変更 → SE級（修正後報告）
  • フォーマット修正等 → PG級（自動許可）

全ての書き込み操作が、実行前に権限チェックされる

PostToolUse — 学習者

  • テスト失敗 → 成功パターンの自動検出
  • tdd-patterns.log への記録（PG級、自動）
  • doc-sync-flag の生成（src/ 変更時）

失敗と成功のパターンを蓄積し、フレームワークの進化に繋げる

Stop hook — ループ制御

  • lam-loop-state.json による状態管理
  • Green State 判定（テスト + lint）
  • 反復回数制限（デフォルト5回）
  • フルスキャン制御（差分 → 全体の二段階）

無限ループを防ぎ、確実に収束させるための安全装置

権限等級 PG / SE / PM

等級意味アクション例
PG 自明な修正 自動修正可 typo, import整理, lint修正
SE 技術判断あり 修正後報告 テスト追加, 内部リファクタ
PM 仕様影響あり 承認必須 API変更, ルール変更, 仕様変更

PG で迷ったら SE級に丸める（安全側に倒す）

Green State 5条件

#条件内容
G1テスト全パスpytest / npm test 等
G2lint エラーゼロruff / eslint 等
G3対応可能 Issue ゼロPG/SE級は修正済み
G4仕様差分ゼロdocs/specs/ と実装の整合性
G5セキュリティ通過依存脆弱性 + シークレットスキャン

Green State = スキャンで Issue が 0件。修正後に 0件ではない

MAGI System

MELCHIOR

科学者（推進者）

  • Value
  • Speed
  • Innovation

BALTHASAR

母（批判者）

  • Risk
  • Security
  • Debt

CASPAR

女（調停者）

  • Synthesis
  • Balance
  • Decision

MAGI: 3つの独立した思考体による合議（エヴァンゲリオン由来）

AoT (Atom of Thought)

適用条件（いずれか該当）

  • 判断ポイントが 2つ以上
  • 影響レイヤーが 3つ以上
  • 選択肢が 3つ以上

複雑な判断を独立した Atom に分解し、
各 Atom で MAGI 合議 → Reflection で結論検証
