PLANNING ADR と設計

Living Architect

MAGI System で技術選定を検討します。MELCHIOR（科学者/推進者）、BALTHASAR（母/批判者）、CASPAR（女/調停者）が合議します。

MELCHIOR（推進者）

  • FastAPI + JWT で高速
  • Stripe SDK が充実
  • Redis でセッション管理

BALTHASAR（批判者）

  • JWT 即時失効が困難
  • Stripe webhook 検証必須
  • 個人情報保護の考慮

CASPAR（調停者）

  • JWT + Redis で即時失効対応
  • webhook 署名検証を必須化
  • ADR-0001 として記録

PLANNING 設計ドキュメント出力

## docs/specs/auth-design.md ### 認証アーキテクチャ - 方式: JWT (access + refresh token) - アクセストークン有効期限: 15分 - リフレッシュトークン: Redis 管理、7日有効 - パスワード: bcrypt (cost=12) ### エンドポイント - POST /auth/login - POST /auth/logout - POST /auth/refresh - POST /auth/password-reset

BUILDING TDD: Red

## tests/test_auth.py def test_login_returns_jwt_token(): response = client.post("/auth/login", json={ "email": "user@example.com", "password": "secure123" }) assert response.status_code == 200 assert "access_token" in response.json()

Red: まずテストを書く。まだ実装がないので失敗する

BUILDING TDD: Green

## src/auth/router.py @router.post("/auth/login") async def login(credentials: LoginRequest): user = await authenticate_user(credentials) token = create_access_token(user.id) return {"access_token": token}

Green: テストが通る最小限の実装
